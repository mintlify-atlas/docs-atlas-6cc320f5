This guide walks you from installation to a running app with a fully generated CRUD resource.

## Prerequisites

- Go 1.24 or later
- PostgreSQL running locally or accessible remotely
- Linux or macOS (Windows is not supported)

## Create your app

<Steps>
  <Step title="Install the Andurel CLI">
    ```bash
    go install github.com/mbvlabs/andurel@v1.0.0-beta.3
    ```

    Verify the installation:

    ```bash
    andurel --version
    ```
  </Step>

  <Step title="Create a new project">
    ```bash
    andurel new myapp
    cd myapp
    ```

    To add extensions at creation time:

    ```bash
    # With Docker support
    andurel new myapp -e docker

    # With AWS SES email
    andurel new myapp -e aws-ses

    # Multiple extensions
    andurel new myapp -e docker,aws-ses
    ```
  </Step>

  <Step title="Install managed tools">
    ```bash
    andurel tool sync
    ```

    This downloads the project's tool binaries (goose, sqlc, templ, shadowfax) into `bin/` according to the versions pinned in `andurel.lock`.
  </Step>

  <Step title="Configure your database">
    ```bash
    cp .env.example .env
    ```

    Edit `.env` with your PostgreSQL connection details:

    ```bash .env
    DB_KIND=postgres
    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=myapp_development
    DB_USER=postgres
    DB_PASSWORD=your_password
    DB_SSL_MODE=disable
    ```

    If the database doesn't exist yet, create it:

    ```bash
    andurel database create
    ```
  </Step>

  <Step title="Run migrations">
    ```bash
    andurel migrate up
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    andurel run
    ```

    Your app is now running at `http://localhost:8080` with live reload enabled for Go code, templates, and CSS.
  </Step>
</Steps>

## Generate your first resource

Resources are the heart of Andurel. A single command generates a complete CRUD interface.

<Steps>
  <Step title="Create a migration">
    ```bash
    andurel migrate new create_products_table
    ```

    This creates a new SQL file in `database/migrations/`. Edit it to define your table:

    ```sql database/migrations/TIMESTAMP_create_products_table.sql
    -- +goose Up
    -- +goose StatementBegin
    CREATE TABLE products (
        id          uuid PRIMARY KEY DEFAULT gen_random_uuid(),
        created_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now(),
        updated_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT now(),
        name        VARCHAR(255) NOT NULL,
        description TEXT,
        price       NUMERIC(10, 2) NOT NULL DEFAULT 0
    );
    -- +goose StatementEnd

    -- +goose Down
    -- +goose StatementBegin
    DROP TABLE IF EXISTS products;
    -- +goose StatementEnd
    ```

    <Warning>
      Your table must have an `id` primary key. Supported types: `uuid`, `serial`, `bigserial`, or `varchar`/`text` (string). The generator reads the schema to wire everything up correctly.
    </Warning>
  </Step>

  <Step title="Apply the migration">
    ```bash
    andurel migrate up
    ```
  </Step>

  <Step title="Generate the resource">
    ```bash
    andurel generate resource Product
    ```

    This single command creates:

    | File | Description |
    |------|-------------|
    | `models/product.go` | Model with CRUD functions |
    | `models/factories/product_factory.go` | Factory for test data |
    | `database/queries/products.sql` | SQLC query definitions |
    | `controllers/products_controller.go` | Controller with all CRUD actions |
    | `router/routes/products.go` | Typed route variables |
    | `router/connect_products_routes.go` | Route registration |
    | `views/products_*.templ` | Templ view templates |
  </Step>

  <Step title="Reload and visit your resource">
    The dev server auto-reloads. Visit `http://localhost:8080/products` to see your new CRUD interface.
  </Step>
</Steps>

<Tip>
  Resource generation is a one-time scaffold. The generated code is entirely yours to own and modify. Andurel won't overwrite it — you manage it going forward.
</Tip>
