## Why Andurel?

Andurel eliminates boilerplate and lets you focus on building features. One command creates a full CRUD resource — model, controller, views, routes, and SQL queries all wired together.

## Get started in minutes

<Steps>
  <Step title="Install Andurel">
    ```bash
    go install github.com/mbvlabs/andurel@v1.0.0-beta.3
    ```
  </Step>
  <Step title="Create a new project">
    ```bash
    andurel new myapp
    cd myapp
    andurel tool sync
    ```
  </Step>
  <Step title="Configure your database">
    ```bash
    cp .env.example .env
    # Edit .env with your PostgreSQL connection details
    andurel migrate up
    ```
  </Step>
  <Step title="Start developing">
    ```bash
    andurel run
    ```
    Your app is now running at `http://localhost:8080` with live reload enabled.
  </Step>
</Steps>

## Core technologies

Andurel is built on a carefully selected stack of proven Go libraries:

| Technology | Purpose |
|---|---|
| [Echo](https://echo.labstack.com/) | High-performance HTTP router |
| [SQLC](https://sqlc.dev/) | Type-safe SQL code generation |
| [Templ](https://templ.guide/) | Type-safe HTML templates |
| [Datastar](https://data-star.dev/) | Hypermedia-driven frontend interactivity |
| [River](https://riverqueue.com/) | PostgreSQL-backed background jobs |
| [OpenTelemetry](https://opentelemetry.io/) | Logs, traces, and metrics |
| [Tailwind CSS](https://tailwindcss.com/) | Utility-first CSS |

<Note>
  Andurel currently supports **Linux** and **macOS** only. Windows support is not available yet. See [issue #382](https://github.com/mbvlabs/andurel/issues/382) if you'd like to contribute Windows support.
</Note>
