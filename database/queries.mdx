title: "Queries"
description: "Write SQL queries and let SQLC generate type-safe Go code using the andurel query command."
---

Andurel uses [SQLC](https://sqlc.dev/) for type-safe database access. You write SQL query files in `database/queries/`, and SQLC generates the corresponding Go functions and types into `models/internal/db/`.

<Note>
  Never edit the files in `models/internal/db/` by hand — they are fully generated by SQLC and will be overwritten the next time you run `andurel query compile`.
</Note>

## Commands

| Command | Description |
|---|---|
| `andurel query generate [table_name]` | Generate SQLC queries for a table |
| `andurel query refresh [table_name]` | Refresh queries for a table to match the current schema |
| `andurel query compile` | Compile all queries and regenerate Go code |
| `andurel query validate` | Validate `database/sqlc.yaml` against framework requirements |

The `query` command also has the alias `q`.

### generate vs. refresh

- **`generate`** — creates a new query file at `database/queries/{table_name}.sql` for a table that does not yet have one. Returns an error if the table is not found in the migrations.
- **`refresh`** — updates an existing query file to reflect the current table schema. Use this after adding or removing columns.

```bash
# Generate queries for a new junction table
andurel query generate user_roles

# Keep an existing query file in sync after a schema change
andurel query refresh user_roles
```

## Query file format

SQL query files use SQLC annotation comments to name each query and declare its return type.

```sql database/queries/products.sql
-- name: GetProduct :one
SELECT * FROM products
WHERE id = $1;

-- name: ListProducts :many
SELECT * FROM products
ORDER BY created_at DESC;

-- name: CreateProduct :one
INSERT INTO products (
    name,
    description,
    price_cents
) VALUES (
    $1, $2, $3
)
RETURNING *;

-- name: UpdateProduct :one
UPDATE products
SET
    name        = $1,
    description = $2,
    price_cents = $3,
    updated_at  = NOW()
WHERE id = $4
RETURNING *;

-- name: DeleteProduct :exec
DELETE FROM products
WHERE id = $1;
```

### Return type annotations

| Annotation | Description |
|---|---|
| `:one` | Returns a single row |
| `:many` | Returns a slice of rows |
| `:exec` | Executes the statement, returns no rows |

## SQLC configuration

The SQLC configuration lives at `database/sqlc.yaml`. Andurel manages a base configuration internally; `database/sqlc.yaml` is a user overlay that merges on top of it.

You can add custom `gen` options to the overlay, but the base settings — package name, output path, SQL package, and UUID type override — are fixed by the framework and validated by `andurel query validate`.

```bash
# Check that your sqlc.yaml satisfies framework requirements
andurel query validate
```

## Compiling queries

After adding or editing any `.sql` file in `database/queries/`, run:

```bash
andurel query compile
```

This runs `sqlc compile` (checks for errors) followed by `sqlc generate` (regenerates `models/internal/db/`) in sequence. The SQLC binary is managed by Andurel at `bin/sqlc`.

<Tip>
  Run `andurel query compile` as part of your workflow any time you change a migration or a query file, so the generated Go types stay in sync with the database schema.
</Tip>
